      到此，博客引擎已经基本开发完毕了。当我们需要将产品向全球市场推广 时，就必须进行本地化工作。在实际应用中，通常将国际化和本地化操作安排在项目功能开发完成之后再进行(当然也可以安排在项目的开始阶段)。下面以博客引 擎为例，进行应用的国际化和本地化操作演示。


注意：

在项目功能开发过程中，采用统一的语言。当开发完毕后，再按照需求配置不同的本地化语言。


12.12.1 国际化和本地化#

      本地化和国际化的国际组织是Openil18n，其前身是li18nux。下面给出国际化和本地化的基本定义：

国际化：软件国际化(I18N，Internationalization的简写——由于首字母“I”和末尾字母“n”间有18个字 符，所以简称I18N)是指所开发的软件能适应全世界市场的本地化工作，但又不需要对程序做任何系统性或结构性变化的特征。这种特征通过特定的系统设计， 程序设计，编码方式来实现。从编程的角度考虑，就是在应用中避免使用具有本地化特色的代码。因此在应用程序中，需要通过模版来为不同的使用者显示不同的视 图界面(其中模板显示的样式通过messages文件控制)。国际化还包括了不同数据类型的显示规范，比如时间，货币，数字等。

本地化：软件本地化(L10N，英文的Localization一词的缩写。由于首字母“L”和末字母“n”间有10个字母，所以简 称L10N)是指将一个软件产品按照特定地区的需要而进行定制的过程，即在原语言版本的基础上，通过翻译、定制和参数配置等工作，使软件产品或系统在语 言、时区等各个方面与当地的相应内容相一致。即为应用准备一个具有本地化特色的服务，方便应用“入乡随俗”。因此一个应用(已经具备国际化)往往包含一个 或多个本地化版本。在应用程序中，将使用本地语言的视图界面(通过messages文件控制模板)显示给用户。浏览器会自动为不同地区的用户选择最佳的本 地化版本(前提是该应用具有本地化特征)。

      国际化与本地化是一个辩证的关系，两者缺一不可。国际化是核心，是内在的实 现，是将来本地化的基础，使本地化过程不需要对代码改动就能完成。另一方面，本地化是外在的表现，通过定制、配置等工作实现本地化。


12.12.2 为yabe博客引擎配置本地化#

      打开conf/application.conf文件，增加如下配置(或者去掉该配置前注释符#)，以此来支持三种本地化语言(English，Dutch和French)： 

# Localisations for English, Dutch and French. 
application.langs=en,nl,fr
      完成以上配置后启动应用，Play会发出以下警告。这是因为我们还没有配置相应的本地化messages文件： 
16:19:04,728 WARN ~ Messages file missing for locale en 
16:19:04,729 WARN ~ Messages file missing for locale nl 
16:19:04,729 WARN ~ Messages file missing for locale fr


注意：

本节的代码可以在Play框架的samples-and-tests/yabe目录下找到。本例中以法语(French)和荷兰语(Dutch)作为本地化语言，实现应用程序的国际化。


12.12.3 定义messages文件#

      在conf/目录下创建以下三个文件，替代原有的messages文件，就可以解决上面出现的问题了：

  • messages.en
  • messages.nl
  • messages.fr

      在此我们需要注意：messages文件和Java中的属性文件(properties files)使用相同的语法规则，但编码格式却是不一样的。messages文件为为纯文本文件，采用UTF-8作为编码格式，用于设置本地化语言，以下是希腊语的本地化例子： 

hello.morning = \u0152\u222b\u0152\u00b1\u0152\u00aa\u0152\u2211\u0152\u00ba\u0152\u2260\u0153\u00c5\u0152\u00b1
hello.informal = \u0152\u2265\u0152\u00b5\u0152\u03c0\u0152\u00b1\u0153\u00c9\u0152\u00f8\u0153\u00d6
      我们也可以在定义中直接使用希腊字母： 
hello.morning = καλημέρα 
hello.informal = γεια σου


注意：

Java中的属性文件采用ISO-8859-1或Latin-1作为编码格式。


12.12.4 编写简单的messages文件#

      本例针对yabe/app/views/main.html模版进行本地化操作，需要本地化操作的对象是字符串“Log in to write something”： 

<ul id="tools"> 
    <li> 
        <a href="@{Admin.index()}">Log in to write something</a> 
    </li> 
</ul>

      为了实现国际化需求，使用&{}标签替换原字符串： 

<ul id="tools"> 
    <li> 
        <a href="@{Admin.index()}">&{'views.main.tools.login'}</a> 
    </li> 
</ul>

      之后在定义的三个messages文件中加入对应的本地化描述：

conf/messages.en： 

views.main.tools.login = Log in to write something
conf/messages.nl： 
views.main.tools.login = Inloggen om iets te schrijven
conf/messages.fr： 
views.main.tools.login = Connectez-vous pour écrire quelque chose


注意:

&{'key'}标签中的key的值是可以任意设定的，只需要在对应的messages本地化文件中定义即可。


      当我们保存了这些设置后，就可以在浏览器中通过改变语言设定来查看不同语言版本的界面了(设置不同的Accept-Language HTTP请求头)。


补充:

Firefox浏览器的设置方法如下：工具 » 选项 » 内容 » 语言 »选择。如果列表中没有该语言选项则通过添加按钮来导入French\[fr] 或 Dutch\[nl]（如图12.41所示），完成后重启浏览器生效。


(图12.41 Firefox浏览器中设置语言)


12.12.5 本地化模型对象#

      当我们登录到博客引擎的管理域页面时(http://localhost:9000/admin)，会看到被管理的模型对象列表(posts，tags，comments，users)，以及由CRUD模块提供的相应的模型编辑页面。这些页面中的标题默认采用域模型命名(也就是JavaBean的名字或属性名称)。

      CRUD模块已经具备了国际化支持，只需在messages本地化文件中使用JavaBean或属性名称作为key值进行配置即可。编辑conf/messages.nl文件，进行如下配置： 

post = artikel 
Post = Artikel 
posts = artikelen 
Posts = Artikelen 
comment = reactie 
Comment = Reactie 
comments = reacties 
Comments = Reacties 
user = gebruiker 
User = Gebruiker 
users = gebruikers 
Users = Gebruikers
      编辑conf/messages.fr文件，进行如下配置： 
post = article 
Post = Article 
posts = articles 
Posts = Articles 
comment = commentaire 
Comment = Commentaire 
comments = commentaires 
Comments = Commentaires 
user = utilisateur 
User = Utilisateur 
users = utilisateur 
Users = Utilisateurs

      刷新浏览器查看结果。编辑页面都已经实现了本地化操作，但是导航栏还没有改变，如图12.42所示：

(图12.42 导航栏)

      这是因为导航栏是我们在views/admin.html中自己定义的。使用&{'...'}标签将其替代，实现本地化操作： 

<a href="@{Posts.list()}">&{'Posts'}</a> 
...
<a href="@{Tags.list()}">&{'Tags'}</a> 
...
<a href="@{Comments.list()}">&{'Comments'}</a> 
...
<a href="@{Users.list()}">&{'Users'}</a>


12.12.6 定义带参数的messages文件#

      本地化messages文件中也可以定义参数。如果本地化操作中包含单个参数变量，可以按照如下方式进行描述(与Java语法非常类似)： 

views.Application.listTagged.title = Posts tagged with %s

      模板中以如下方式使用，传递单个参数变量： 

&{'views.Application.listTagged.title', tag}

      当本地化messages文件中包含多个参数变量时，需要引入索引机制： 
views.Admin.index.welcome = Welcome %1$s, <span>you have written %2$s posts so far</span>

      模板中以如下方式使用，传递多个参数变量： 

&{'views.Admin.index.welcome', user, posts.size()}

      在博客引擎的例子中，我们希望文章post以正确的单复数形式显示，因此可以将post是否为复数也作为一个参数进行传递： 
views.Admin.index.welcome = Welcome %1$s, <span>you have written %2$s %3$s so far</span>

      在模板中使用pluralize扩展判断文章是否为复数： 

&{'views.Admin.index.welcome', user, posts.size(), posts.pluralize(messages.get('post'), messages.get('posts'))}  

补充：

messages.get()方法用于参照messages文件中的本地化描述是否为复数。

12.12.8 Play本地化操作的优势#

      具有开发经验的程序员可能会发现，传统的本地化操作非常笨拙。以JavaServer Faces框架为例，介绍该框架本地化操作的一些设计缺陷：

  1. 无法在属性中传递本地化参数。
  2. 参数格式化非常繁琐。
  3. 无法实现超链接的本地化操作。

      Play的设计师很巧妙地避免了这些问题，可以非常简单地处理这些特殊情况。


补充：

JavaServer Faces(JSF)是一种标准的J2EE表示层的技术，其主旨是为了使Java开发人员能够快速的开发基于Java的Web应用程序。它不同于其它Java表示层技术的最大优势是其采用了组件模型，事件驱动。


      两种框架针对问题一的解决方案对比：

      我们希望在模板的<a>标签的title属性中传递参数： 

<a href="@{Application.show(_post.id)}" title="By Bob">

      这个问题在JSF中尤为明显,JSF通常使用XML标签代替参数，但属性是不支持使用标签的。Play语法就可以很好地避免这个问题： 

<a href="@{Application.show(_post.id)}" title="&{'views.tags.display.author', _post.author.fullname}">

      两种框架针对问题二的解决方案对比：

      如果在JSF中需要对参数进行格式化(比如格式化日期参数2009-06-14)， 必须使用XML标签，这同样会在属性操作时产生问题。让我们来看下在Play中是如何解决的： 

<span>&{'views.tags.display.author', _post.author.fullname, comment.postedAt.format('yyyy-MM-dd')}"}</span>

      当然也可以按照以下方式实现参数的格式化： 

<span>&{'views.tags.display.author', _post.author.fullname, comment.postedAt.format(messages.get('views.dateFormat'))}"}</span>


      两种框架针对问题三的解决方案对比：

      当需要本地化操作的内容是带有超链接时(以Log in to write something为例)，JSF会产生很多问题。因为超链接是JSF的组件，链接标签(<a></a>)无法在本地化文件中进行描述。Play以纯HTML文件作为模板，可以直接在本地化操作中描述标签，并将超链接的URL作为参数进行传递： 

logIn = <a href="%s">Log in</a> to write something 
&{'logIn', '/admin'}

      当“/admin”参数传入本地化文件的logIn描述后，根据路由规则生成如下超链接： 

<a href="@{Admin.index()}">
