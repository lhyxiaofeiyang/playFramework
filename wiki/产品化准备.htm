12.11.1 定义框架 ID#

      实际项目通常会在不同的计算机（服务器）上部署应用程序。Play允许为每个安装好的框架指定唯一ID，通过application.conf文件进行统一管理。


补充：

实际项目中，产品的开发环境与部署环境通常就不会配置在同一台设备上。


      如果需要将产品化程序部署在server01服务器，需要在该服务器上安装Play框架，之后使用play id命令设置框架ID： 

play id server01

      为框架指定ID后，就可以在应用的application.conf文件中定义不同配置，对运行服务器上的程序进行管理。

12.11.2 指定PROD模式#

      配置文件中的application.mode属性用于指定应用的运行模式。以服务器server01为例，配置yabe/conf/application.conf文件： 

%server01.application.mode=PROD

      之后当Play应用运行在server01服务器上时，程序就会自动以PROD模式启动。


提示：

如果产品还处于开发阶段，推荐使用DEV模式，因为在DEV模式下Play采用热加载模式（hot reload），自动编译Java源文件，并且可以准确定位错误代码块。应用部署时请采用PROD模式，因为该模式下启动Play后会自动编译所有的 Java源文件和模板文件，之后将不再检查Java源文件和模板是否被更改，效率更高。


12.11.3 配置MySQL服务器#

      运行在内存中的HSQLDB显然不能满足实际项目的需求，本节介绍如何在Play应用中使用MySQL数据库。


提示：

Play集成了MySQL的JDBC驱动程序，所以我们不需要手动添加相应的JAR包。


      在yabe/conf/application.conf文件中配置数据源： 

%server01.db=mysql:root:secret@yabe
      就完成了MySQL数据库的配置。


12.11.4 配置JPA管理模式#

      Hibernate具有自动管理数据库的特性。即Java模型对象有改变时，Hibernate会自动将改变的数据同步到数据库中。具体配置如下： 

%server01.jpa.ddl=update

      显然以该模式运行Play产品并不可行，因为程序的行为将变得不可预见 (Hibernate自动修改数据库表结构和数据)。如果我们不想让Hibernate自动同步数据到数据库，可以将jpa.ddl的值修改为 validate： 

%server01.jpa.ddl=validate


提示：

DEV模式默认采用update配置。


12.11.5 设置frontal HTTP服务器 #

      将应用的HTTP端口设置为80，就基本完成部署服务器的架设了。 

%server01.http.port=80
      默认的设置只允许在给定的服务器上安装一个Play应用。如果需要在同一台服务器上部署多个应用程序（使用不同的IP主机名），可以使用HTTP服务器作为逆向代理。

      Play兼容任何HTTP服务器作为逆向代理，lighttpd就是一个不错的选择。关于lighttpd的配置已经超出了本书的范围，下面只给出主要配置： 

server.modules = (
      "mod_access",
      "mod_proxy",
      "mod_accesslog" 
)
...
$HTTP["host"] =~ "www.yabe.com" {
    proxy.balance = "round-robin" proxy.server = ( "/" =>
        ( ( "host" => "127.0.0.1", "port" => 9000 ) ) )
}

      本地逆向代理可以通过读取application.conf中的配置连接到Play应用程序： 

%server01.XForwardedSupport=127.0.0.1


补充：

lighttpd是一种轻量快速的HTTP服务器。
