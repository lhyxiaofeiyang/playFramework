      Secure模块管理应用配置基本的验证和认证。该模块提供了通用的controllers.Secure控制器，里面定义了一系列拦截器，使用@with注解可以很容易地将它们添加到所需的控制器中。


11.4.1 启用Secure模块#

      Secure是Play内置的模块，不用进行安装，只需在/conf/application.conf文件中添加如下配置，就可以将其引入： 

# The secure module
module.secure=${play.path}/modules/secure


11.4.2 导入默认Secure路由#

      配置/conf/routes文件，引入Secure模块默认路由： 

# Import Secure routes
*      /                module:secure


注意：

并不是只能采用默认的路由配置，还可以自定义路由或者两者混用。


11.4.3 控制器保护#

      在应用中引入Secure模块，就可以使用@With注解保护控制器： 

@With(Secure.class)
public class Application extends Controller {
    
    public static void index() {
        render();
    }
}

      上例代码为Application控制器添加了@With(Secure.class)注解，这样Secure模块的默认认证页面就会自动保护该控制器了。


11.4.4 自定义认证机制#

      如果不定义认证机制，登录页面会接受任何形式的用户名和密码。在自定义认证方面，模块提供的Secure.Security辅助类使得针对应用的安全开发更加便利。只需要在controllers包中创建继承于Secure.Security的类，并覆盖authentify（String username,String password）方法。以下为具体实现： 

package controllers;
 
public class Security extends Secure.Security {
    
    static boolean authenticate(String username, String password) {
        User user = User.find("byEmail", username).first();
        return user != null && user.password.equals(password);
    }
}


注意：

还可以覆盖其他的方法(onAuthenticated, onDisconnected)，进行自定义验证。


11.4.5 获取已连接用户#

      在应用的其他代码中，可以重用刚创建的Security辅助类获取已连接的用户。 

@With(Secure.class)
public class Application extends Controller {
    
    public static void index() {
        String user = Security.connected();
        render(user);
    }
    
}


11.4.6 添加认证检查#

使用注解验证

      @Check注解会通知Secure模块检查建立连接的用户是否具有调用权限： 

@With(Secure.class)
public class Application extends Controller { 
   ...
   
   @Check("administrator")
   public static void delete(Long id) {
       ...
   }
}

      如果控制器中所有的Action都需要进行认证检查，可以直接将@Check注解添加至控制器上： 

@With(Secure.class)
@Check("administrator")
public class Application extends Controller { 
   ...
   
   public static void delete(Long id) {
       ...
   }
   
   public static void edit(Long id) {
       ...
   }
}

      Secure模块默认会使用所有的验证方法，可以通过覆盖一个或多个Security类中的方法进行定制，提高验证效率： 

package controllers;
 
public class Security extends Secure.Security {
    ...
    
    static boolean check(String profile) {
        User user = User.find("byEmail", connected()).first();
        if ("administrator".equals(profile)) {
            return user.admin;
        }
        else {
            return false;
        }
    }    
}


使用标签验证

      在视图模板中，可以以标签的形式使用Secure安全认证： 

#{secure.check "administrator"}
    <a href="@{delete(item.id)}">Delete</a>
#{/secure.check}
      上例代码中，只有当用户通过管理员认证后，模板才会显示Delete链接。
