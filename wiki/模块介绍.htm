      Play应用程序可以由多个应用模块组装而成。这使得可以将大型应用分割为多个较小的应用模块，从而提高应用程序组件的重用性。


11.1.1 模块是什么？#

      Play中的模块可以被理解为应用程序，但两者在资源加载方式上存在一些区别：

  • 模块不存在conf/application.conf配置文件。
  • 模块拥有自己的conf/routes配置文件，但该路由配置文件不会被自动加载。
  • 应用启动时，Play首先在主应用的目录下搜索，之后才会到所有已加载的模块目录下搜索。
  • 模块的module/lib目录下可以存放JAR格式的第三方库文件。
  • 模块包含文档文件。
  • 在模块中一切都是可选的。


      读者也可以创建自己的模块，只需要使用Play提供的play new-module命令即可。


11.1.2 如何在程序中加载模块#

      框架中模块的默认安装路径为$PLAY_HOME/modules，该目录下的模块可以供所有应用程序使用。如果需要在应用程序中加载外部模块，需要配置应用程序的application.conf文件，引入模块： 

# Additional modules
# ~~~~~
# A module is another Play application. Add a line for each module you want
# to add to your application. Module paths are either absolute or relative to
# the application root.
#
module.crud=${play.path}/modules/crud

      读者可以按照自己的需求，选择性地引入模块。比如某个大型应用程序包含CMS模块、FORUM模块、DIRECTORY模块和GWT模块。首先，可以按照以下目录结构创建项目：

/my-project

  • /cms
  • /forum
  • /directory
  • /gwt
  • /main


      其中main是主程序目录（使用play new命令创建），GWT模块需要从模块库安装，CMS模块、FORUM模块以及 DIRECTORY模块均由play new-module命令创建。


补充：

GWT即Google Web Toolkit的缩写。有了GWT可以使用Java编程语言编写Ajax前端，然后GWT会交叉编译到优化的JavaScript中，而JavaScript可以自动在所有主要浏览器上运行。GWT允许开发人员使用Java编程语言快速构建和维护复杂但性能高的JavaScript前端应用程序，从而降低了开发难度，尤其是与Eclipse Google插件结合使用时，优势更明显。


      之后配置主程序文件（my-project/main/conf/application.conf），引入所需的模块： 

# Additional modules
# ~~~~~
mdoule.gwt=../gwt
module.cms=../cms
module.forum=../forum
module.directory=../directory

      当主程序启动时(使用play run my-project/main命令)，Play会将所有需要的模块加载至内存中。


注意：

如果以相对路径配置模块，Play会相对主程序根目录解析目的地址。


11.1.3 加载模块路由#

      每个模块都包含自身的路由文件。需要在主程序的routes文件中进行配置，才能加载模块的路由： 

# Import the default CRUD routes
GET     /admin	     module:crud

      也可以直接加载所有的可用模块路由： 

GET     /	     module:*


11.1.4 添加模块文档#

      Play为模块配备了文档功能。可以为模块动态添加文档，记录模块的信息或者使用说明，方便日后开发者查询。Play文档采用特定的语法规则编写，并以.textile格式保存在$PLAY_HOME/documentation/manual/目录下。如果为Play添加了新的文档，需要在$PLAY_HOME/documentation/manual/home.textile文件中为其添加相应的目录。

      如果正在运行的Play应用引入了附带文档的模块，那么访问http://localhost:9000/@documentation就可以查看文档的目录页面。


补充：

Textile是人性化的Web文本生成器，以简洁的方式提供HTML标签功能，使用类似wiki的语法生成HTML格式化的文档内容。


11.1.5 使用模块库#

      Play模块库中包含的所有模块均由官方社区提供。一个模块可能会有多个不同版本，所以使用者需要事先阅读文档查看模块的版本变更情况，选择适合自己框架的模块。

      也可以使用play list-modules命令来浏览模块库，以下是使用play list-modules命令的结果： 

gbo-mac:~ guillaume$ play list-modules

~
~ [cobertura]
~   Cobertura
~   http://www.playframework.org/modules/cobertura
~   Versions: 1.0
...

      play install {module}-{version}是模块安装命令。例如可以使用play install命令安装支持Scala的模块： 

play install scala-head

      按照惯例， 以head标识的模块是不稳定的版本。可以在命令中省略该标识，安装默认版本的模块： 

play install scala

      play install命令默认将模块安装至$PLAY_HOME/modules/目录。可以在该命令中配置--path选项更改模块的安装路径： 

play install gwt --path=my-project


11.1.6 提交新的模块到模块库#

      用户也可以将自己创建的模块提交给Play官方社区。模块提交者需要有自己的OpenID，并通过Google Group向Play官方发送注册模块的申请。Play社区会根据OpenID验证模块作者，并要求模块作者提供更加详细的模块信息，具体内容如下：

  • 模块的作用。
  • 模块名称，命名必须符合 [a-zA-Z]+ 正则表达式。
  • 模块的简短描述。
  • 模块提交者的项目主页。
  • 模块提交者的OpenID。
  • 提交的模块需要托管在可以存放源代码和反馈Bug的平台。如果提交者不知道有什么好的平台，github，Google Code和Launchpad都是不错的选择。

补充：

Github：Git是分布式的版本控制系统，最初由Linus Torvalds编写，用作Linux内核代码的管理。GitHub可以托管各种Git库，并提供Web界面。

Google Code是一个开源社区，它提供了svn、cvs等多种代码管理方式，用于存放开发人员的工具和API技术资源。而且Google Code不支持私有项目，即所有项目全部开源，任何人都可以获得项目的所有代码。

Launchpad是Ubuntu母公司Canonical有限公司所资助架设的网站，是一个提供维护、支援或连络Ubuntu开发者的平台。在Launchpad网站上可以进行Ubuntu（或其相关软件）的翻译工作或管理开发专权，也可以利用该网站的回报机制来回报相关软体的Bug，或进一步提供相关建议。


      如果通过审核，模块作者可以使用play build-module命令发布模块，Play会自动连接到模块库并上传生成的包。更多有关模块的讨论都在Google Group，欢迎大家分享Play开发经验。
